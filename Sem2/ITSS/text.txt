						<h1 class="entry-title">AWK command in Unix/Linux with examples</h1>
				
						</header><!-- .entry-header -->

				<div class="entry-content">
			<p>Awk is a scripting language used for manipulating data and generating reports.The awk command programming language requires no compiling, and allows the user to use variables, numeric functions, string functions, and logical operators.</p>
<p>Awk is a utility that enables a programmer to write tiny but effective programs in the form of statements that define text patterns that are to be searched for in each line of a document and the action that is to be taken when a match is found within a line. Awk is mostly used for pattern scanning and processing. It searches one or more files to see if they contain lines that matches with the specified patterns and then performs the associated actions.</p>
<p>Awk is abbreviated from the names of the developers &#8211; Aho, Weinberger, and Kernighan.</p>
<p><strong>WHAT CAN WE DO WITH AWK ?</strong></p>
<p><strong>1. AWK Operations:</strong><br />
(a) Scans a file line by line<br />
(b) Splits each input line into fields<br />
(c) Compares input line/fields to pattern<br />
(d) Performs action(s) on matched lines</p><br/>
        <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>
          <!-- post_top_responsive -->
          <ins class="adsbygoogle"
          style="display:block"
          data-ad-client="ca-pub-9465609616171866"
          data-ad-slot="4501693235"
          data-ad-format="auto"></ins>
          <script>
          (adsbygoogle = window.adsbygoogle || []).push({});
        </script>
        <br/>
            
<p><strong>2. Useful For:</strong><br />
(a) Transform data files<br />
(b) Produce formatted reports</p>
<p><strong>3. Programming Constructs:</strong><br />
(a) Format output lines<br />
(b) Arithmetic and string operations<br />
(c) Conditionals and loops</p>
<p><strong>Syntax:</strong></p>
<pre>
<strong>awk options 'selection _criteria {action }' input-file &gt; output-file</strong>
</pre>
<p><strong>Options:</strong></p>
<pre>
-f program-file : Reads the AWK program source from the file 
                  program-file, instead of from the 
                  first command line argument.
-F fs            : Use fs for the input field separator
</pre>
<p>                                           <strong>Sample Commands</strong></p>
<p><strong>Example:</strong><br />
Consider the following text file as the input file for all cases below.</p>
<pre>
$cat &gt; employee.txt </pre>
<pre>
ajay manager account 45000
sunil clerk account 25000
varun manager sales 50000
amit manager account 47000
tarun peon sales 15000
deepak clerk sales 23000
sunil peon sales 13000
satvik director purchase 80000 </pre>
<p><strong>1. Default behavior of Awk : </strong> By default Awk prints every line of data from the specified file.</p>
<pre>$ awk '{print}' employee.txt</pre>
<p><strong>Output:</strong></p>
<pre>
ajay manager account 45000
sunil clerk account 25000
varun manager sales 50000
amit manager account 47000
tarun peon sales 15000
deepak clerk sales 23000
sunil peon sales 13000
satvik director purchase 80000 </pre>
<p>In the above example, no pattern is given. So the actions are applicable to all the lines. Action print without any argument prints the whole line by default, so it prints all the lines of the file without failure. </p>
<p><strong>2. Print the lines which matches with the given pattern.</strong></p><br/>
        <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>
          <!-- post_top_responsive -->
          <ins class="adsbygoogle"
          style="display:block"
          data-ad-client="ca-pub-9465609616171866"
          data-ad-slot="4501693235"
          data-ad-format="auto"></ins>
          <script>
          (adsbygoogle = window.adsbygoogle || []).push({});
        </script>
        <br/>
            
<pre>$ awk '/manager/ {print}' employee.txt </pre>
<p><strong>Output:</strong></p>
<pre>
ajay manager account 45000
varun manager sales 50000
amit manager account 47000 </pre>
<p>In the above example, the awk command prints all the line which matches with the ‘manager’.</p>
<p><strong>3. Spliting a Line Into Fields : </strong>For each record i.e line, the awk command splits the record delimited by whitespace character by default and stores it in the $n variables. If the line has 4 words, it will be stored in $1, $2, $3 and $4 respectively. Also, $0 represents the whole line. </p>
<pre>$ awk '{print $1,$4}' employee.txt </pre>
<p><strong>Output:</strong></p>
<pre>
ajay 45000
sunil 25000
varun 50000
amit 47000
tarun 15000
deepak 23000
sunil 13000
satvik 80000 </pre>
<p>In the above example, $1 and $4 represents Name and Salary fields respectively.</p>
<p align="Center"><strong>Built In Variables In Awk</strong></p>
<p>Awk&#8217;s built-in variables include the field variables—$1, $2, $3, and so on ($0 is the entire line) — that break a line of text into individual words or pieces called fields.</p>
<p><strong>NR:</strong> NR command keeps a current count of the number of input records. Remember that records are usually lines. Awk command performs the pattern/action statements once for each record in a file.</p>
<p><strong>NF:</strong> NF command keeps a count of the number of fields within the current input record.</p>
<p><strong>FS:</strong> FS command contains the field separator character which is used to divide fields on the input line. The default is &#8220;white space&#8221;, meaning space and tab characters.  FS can be reassigned to another character (typically in BEGIN) to change the field separator.</p><br/>
        <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>
          <!-- post_top_responsive -->
          <ins class="adsbygoogle"
          style="display:block"
          data-ad-client="ca-pub-9465609616171866"
          data-ad-slot="4501693235"
          data-ad-format="auto"></ins>
          <script>
          (adsbygoogle = window.adsbygoogle || []).push({});
        </script>
        <br/>
            
<p><strong>RS:</strong> RS command stores the current record separator character. Since, by default, an input line is the input record, the default record separator character is a newline. </p>
<p><strong>OFS:</strong> OFS command stores the output field separator, which separates the fields when Awk prints them. The default is a blank space. Whenever print has several parameters separated with commas, it will print the value of OFS in between each parameter.</p>
<p><strong>ORS:</strong> ORS command stores the output record separator, which separates the output lines when Awk prints them. The default is a newline character. print automatically outputs the contents of ORS at the end of whatever it is given to print.</p>
<p><strong>Examples:</strong></p>
<p><strong>Use of NR built-in variables (Display Line Number)</strong></p>
<pre>$ awk '{print NR,$0}' employee.txt </pre>
<p><strong>Output:</strong></p>
<pre>
1 ajay manager account 45000
2 sunil clerk account 25000
3 varun manager sales 50000
4 amit manager account 47000
5 tarun peon sales 15000
6 deepak clerk sales 23000
7 sunil peon sales 13000
8 satvik director purchase 80000 </pre>
<p>In the above example, the awk command with NR prints all the lines along with the line number.</p>
<p><strong>Use of NF built-in variables (Display Last Field)</strong></p>
<pre>$ awk '{print $1,$NF}' employee.txt </pre>
<p><strong>Output:</strong></p>
<pre>
ajay 45000
sunil 25000
varun 50000
amit 47000
tarun 15000
deepak 23000
sunil 13000
satvik 80000 </pre>
<p>In the above example $1  represents Name and $NF represents Salary. We can get the Salary using  $NF , where $NF represents last field. </p><br/>
        <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>

          style="display:block"

        <br/>
            
<p><strong>Another use of NR built-in variables (Display Line From 3 to 6)</strong></p>
<pre>$ awk 'NR==3, NR==6 {print NR,$0}' employee.txt </pre>
<p><strong>Output:</strong></p>
<pre>
3 varun manager sales 50000
4 amit manager account 47000
5 tarun peon sales 15000
6 deepak clerk sales 23000 </pre>
<p align="center"><strong>More Examples</strong></p>
<p><strong>For the given text file:</strong></p>
<pre>
$cat &gt; geeksforgeeks.txt

A    B    C
Tarun    A12    1
Man    B6    2
Praveen    M42    3
</pre>
<p><strong>1) To print the first item along with the row number(NR) separated with ” &#8211; “ from each line in geeksforgeeks.txt:</strong></p>
<pre>
$ awk '{print NR "- " $1 }' geeksforgeeks.txt
</pre>
<pre>
1 - Tarun
2 – Manav    
3 - Praveen
</pre>
<p><strong>2) To return the second row/item from geeksforgeeks.txt:</strong></p>
<pre>
$ awk '{print $2}' geeksforgeeks.txt
</pre>
<pre>
A12
B6
M42
</pre>
<p><strong>3) To print any non empty line if present</strong></p>
<pre>
$ awk 'NF &gt; 0' geeksforgeeks.txt
</pre>
<pre>
0
</pre>
<p><strong>4) To find the length of the longest line present in the file:</strong></p>
<pre>
$ awk '{ if (length($0) &gt; max) max = length($0) } END { print max }' geeksforgeeks.txt
</pre>
<pre>
13
</pre>
<p><strong>5) To count the lines in a file:</strong></p>
<pre>
$ awk 'END { print NR }' geeksforgeeks.txt
</pre>
<pre>
3
</pre>
<p><strong>6) Printing lines with more than 10 characters:</strong></p>
<pre>
$ awk 'length($0) &gt; 10' geeksforgeeks.txt
</pre>
<pre>
Tarun    A12    1
Praveen    M42    3
</pre>
<p><strong>7) To find/check for any string in any column:</strong></p>
<pre>
$ awk '{ if($3 == "B6") print $0;}' geeksforgeeks.txt
</pre>
<p><strong>8) To print the squares of first numbers from 1 to n say 6:</strong> </p>
<pre>
$ awk 'BEGIN { for(i=1;i&lt;=6;i++) print &quot;square of&quot;, i, &quot;is&quot;,i*i; }&#039;
</pre>
<pre>
square of 1 is 1
square of 2 is 4
square of 3 is 9
square of 4 is 16
square of 5 is 25
square of 6 is 36
</pre>
